Crie um frontend completo de alta fidelidade (high-fidelity) para um aplicativo financeiro pessoal focado em motociclistas.

IMPORTANTE:

- NÃO implementar backend.
- NÃO implementar autenticação real.
- NÃO implementar APIs.
- NÃO implementar Supabase.
- NÃO implementar Firebase.
- Utilizar apenas dados mockados.
- O objetivo é criar uma experiência visual extremamente próxima de um aplicativo pronto para produção.

Tecnologias:

- React
- TypeScript
- Vite
- Tailwind CSS
- shadcn/ui
- Lucide Icons
- Recharts para gráficos

Design:

- Mobile First
- Inspirado em Nubank, Inter, C6 e aplicativos modernos
- Dark Mode por padrão
- Visual premium
- Fundo preto grafite (#0F1117)
- Cards em cinza escuro
- Cor principal: Roxo (#7C3AED)
- Cor secundária: Azul (#3B82F6)
- Verde para valores positivos
- Vermelho para despesas
- Cantos arredondados (20px)
- Sombras suaves
- Ícones minimalistas
- Animações suaves
- Layout pensado inicialmente para telas de celular

Criar visualização responsiva de smartphone centralizado na tela.

Nome do aplicativo:

MotoLife

Conceito:

Aplicativo de gestão financeira pessoal integrado ao planejamento financeiro de motociclistas.

O usuário deve conseguir controlar:

- Salário
- Renda extra
- Contas fixas
- Alimentação
- Mercado
- Internet
- Energia
- Água
- Condomínio
- Streaming
- Assinaturas
- Saúde
- Estudos
- Lazer
- Combustível
- Manutenção da moto
- Objetivos financeiros

O aplicativo NÃO é apenas sobre motos.

A moto é uma categoria importante dentro da vida financeira do usuário.
